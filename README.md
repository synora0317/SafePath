# 재난안전 AI 대피안내 - 개인/기관 분리 버전

## 데이터·API 키 모두 필요 없습니다
개인 30명·시설 20곳·대피소 32곳(의료지원 여부 포함)·AWS 관측 27곳·재난시나리오가
전부 `data.py`에 내장되어 있습니다.

## 폴더 구조
```
disaster_app2/
├── app.py             # 화면
├── data.py            # 내장 데이터 (위치목록 LOCATIONS 포함)
├── risk_engine.py     # 판단 규칙 (여기서 로직 대부분을 관리)
├── ui_components.py   # 추천 대피소 카드 UI 렌더링
├── theme.py           # 브랜드 색상·폰트·전역 CSS (SafePath)
├── map_view.py         # folium 지도(마커+경로선) 생성
├── routing.py          # OSRM으로 실제 도보 경로 조회 (실패 시 직선 대체)
└── requirements.txt
```

## 실행 방법
```bash
python -m venv venv

pip install -r requirements.txt
streamlit run app.py
```

## 사용 흐름

### 개인 선택 시
① 현재 위치(서울 중구·종로 일대 16개 동) → ② 재난 유형(호우/태풍/대설/산불/지진)
→ ③ 이동수단(도보/차량/휠체어) → ④ 동행자(없음/어린이/노약자/반려동물)

- **휠체어** 선택 → 자동으로 교통약자 분류 → 경사로·엘리베이터 있는 대피소만 추천
- **반려동물** 선택 → 반려동물 동반 가능 대피소만 추천
- 결과는 개인용 안내문 **1개만** 출력됩니다.

### 기관 선택 시
① 현재 위치 → ② 기관 유형(병원/요양시설/학교/어린이집/장애인시설/공장/기타)
→ ③ 재난 유형 → ④ 대피 인원 → ⑤ 대피 수단

- **병원** 선택 → 의료 지원 가능 대피소만 추천
- **병원/요양시설/장애인시설** → 교통약자 접근성 있는 대피소만 추천
- 결과는 기관용 조치사항 **1개만**, 아래 5개 항목으로 출력됩니다:
  1. 현재 재난 위험도
  2. 기관 주변 위험 요소
  3. 추천 대피소 및 추천 이유
  4. 기관 단체 이동 경로 및 이동 시 주의사항
  5. 대피소 운영 상태(잔여 수용 인원, 개방 여부)

## 지도
folium 기반 지도로 나의 위치·대피소 마커와 대피소까지의 경로선을 보여줍니다.
경로선은 OSRM(공개 데모 서버)으로 실제 도보 경로를 우선 조회하며, 조회 실패 시
직선으로 자동 대체됩니다. 인터넷 연결이 필요합니다 (연결 안 되면 직선으로만 표시).

## 위험 단계는 어떻게 정해지나요?
사용자가 직접 고르지 않고, **실시간 AWS 관측값을 시나리오 임계값과 자동 비교**해서 정합니다.
- 호우: 관측 강수량(mm) 기준
- 태풍: 관측 풍속(m/s) 기준
- **산불: 관측 풍속(m/s)을 산불 시나리오의 '대피 권고 단계'와 매칭** (단, 실효습도 등 다른 확산 요인은 실시간 데이터가 없어 반영되지 않음 — 안내문에 이 한계를 명시)
- 대설·지진: 관련 실시간 관측 데이터가 없어 기본값(관심 단계)으로 표시되고,
  안내문에 데이터 한계가 함께 명시됩니다.

⚠️ 참고: 현재 내장된 AWS 관측값은 대체로 평온한 수치(비/바람 거의 없음)라서,
대부분 '관심' 단계로 나올 수 있습니다. 발표 시연에서 더 극적인 '경계·심각' 단계를
보여주고 싶으시면 말씀해 주세요 — 가정 강도를 선택할 수 있는 옵션을 추가해드릴 수 있습니다.
